## Project Status Definitions
- **Proposed**: Project identified but not yet approved
- **Project Initialization**: Project structure created, planning in progress
- **Requirements Analysis**: Gathering and analyzing project requirements
- **Design Phase**: System design and architecture development
- **Development**: Active implementation work
- **Testing**: Quality assurance and validation
- **Deployment**: Production release preparation
- **Complete**: Project delivered and closed
- **On Hold**: Temporarily suspended
- **Cancelled**: Project terminated

## Project Templates
All projects follow the standard EDPS structure:
- `artifacts/` - Analysis outputs and supporting materials
- `tasks/` - Individual task files in GitHub issue format  
- `main.md` - Project overview and navigation
- `project-plan.md` - Detailed project plan with PERT analysis
- `README.md` - Project documentation guide

## Naming Conventions
- Project folders: `[NN] - [Project Name]`
- Sequential numbering starting from 01
- Business-friendly descriptive names
- No special characters in folder names
